# Biblioteca HTTP API v1.0.0

This is the stable agent-facing contract for the Biblioteca MCP-over-HTTP
gateway. Sister projects and writing apps should integrate against this file,
not against internal Python modules.

## Versioning Policy

The API follows semantic versioning.

- Current version: `v1.0.0`.
- Backward-compatible additions keep the major version at `1`.
- Breaking request or response changes require `docs/api/v2.md`.
- Deprecated v1 fields keep working for at least one minor release after a v2
  document is published.

## Authentication

Production deployments are protected end-to-end by Cloudflare Access. There is
no bearer-token fallback.

Machine clients send Cloudflare Access service-token headers to Cloudflare:

- `CF-Access-Client-Id`
- `CF-Access-Client-Secret`

Cloudflare validates those headers and forwards a signed
`Cf-Access-Jwt-Assertion` header to the gateway. The gateway validates the JWT
signature, expiration, and `aud` claim using:

- `CF_ACCESS_TEAM_DOMAIN`
- `CF_ACCESS_AUD_TAG`

The authenticated principal is selected from
`Cf-Access-Authenticated-User-Email` for humans or
`Cf-Access-Service-Token-Name` for service-token clients.

Missing, malformed, expired, or wrong-audience JWTs return `401`. Cloudflare
policy denials return `403` at the edge before the request reaches the gateway.
Missing gateway Cloudflare settings return `503`.

## Error Model

Gateway errors use this shape when the app generates the response:

```json
{
  "error": {
    "code": "source_timeout",
    "message": "Source lab did not respond before the gateway timeout",
    "request_id": "req_018f7f9a"
  }
}
```

Current FastAPI validation errors may additionally use FastAPI's default
`detail` field. Clients should key behavior off HTTP status and `error.code`
when present.

| Status | Code | Meaning |
|---|---|---|
| `401` | `unauthorized` | Missing or invalid Cloudflare Access JWT. |
| `403` | `forbidden` | Cloudflare Access accepted auth but denied policy. |
| `404` | `not_found` | Unknown route, source, page, or tool. |
| `408` | `source_timeout` | A source did not respond before gateway timeout. |
| `422` | `validation_error` | Request shape or query mode is invalid. |
| `503` | `wiki_lock_timeout` | A wiki write lock could not be acquired. |
| `503` | `all_sources_unavailable` | No configured source can answer the request. |
| `500` | `internal_error` | Unexpected gateway error; include `request_id` in reports. |

## Citation Payload Format

`/mcp/query` returns citations as an array of objects. Current citations are
derived from `QueryResult.context_bundle.citations`; v1 clients should accept
missing optional fields because older corpus records may not have all metadata.

```json
{
  "source": "literature",
  "doc_id": "papers/sample-lfp-001",
  "chunk_index": 3,
  "page": null,
  "quote": "Capacity fade accelerates after high-temperature storage.",
  "confidence": 0.94,
  "ingested_at": "2026-04-28T10:30:00Z",
  "verify_against_source": true,
  "citation_type": "evidence"
}
```

Fields:

- `source`: source-server name, usually `literature` or `lab`.
- `doc_id`: stable source document identifier.
- `chunk_index`: zero-based chunk number when the citation points to a chunk.
- `page`: one-based page number when the citation points to a page.
- `quote`: short supporting excerpt.
- `confidence`: score from `0.0` to `1.0`.
- `ingested_at`: ISO-8601 ingestion timestamp when available.
- `verify_against_source`: `true` when the client should re-open the source
  before using the text in a formal report.
- `citation_type`: `evidence`, `wiki`, or `graph`.

## Endpoints

### `POST /mcp`

Streamable HTTP MCP endpoint for writing apps that use MCP clients directly.
It exposes one v1 tool:

- `query`: arguments match `POST /mcp/query`.

The tool returns the same response payload as `POST /mcp/query` in MCP
`structuredContent`. Cloudflare Access authentication is the same as every other
gateway endpoint. The REST routes below remain available for clients that do not
embed an MCP SDK.

Idempotency: follows `POST /mcp/query` for the `query` tool.

### `GET /mcp/health`

Returns configured sources and the current unavailable-source map. This endpoint
is authenticated because source names reveal deployment topology.

Idempotency: safe to retry.

```json
{
  "sources": ["literature", "lab"],
  "unavailable": {}
}
```

### `GET /mcp/sources`

Returns the source registry without subprocess commands, environment variables,
or filesystem paths.

Idempotency: safe to retry.

```json
{
  "sources": [
    {
      "name": "literature",
      "capabilities": ["tool:get_chunks", "intent:reporting"]
    },
    {
      "name": "lab",
      "capabilities": ["tool:read_page", "tag:sop"]
    }
  ]
}
```

### `POST /mcp/literature/{tool_name}`

Forwards the JSON request body as MCP tool parameters to the `literature` source.

Idempotency: read tools such as `read_page`, `list_pages`, and `search_chunks`
are safe to retry. Write tools are not replay-protected in v1.0.0; clients must
not automatically retry write calls unless they own their own idempotency key.

Request:

```json
{
  "query": "LFP degradation",
  "n_results": 5
}
```

Response:

```json
{
  "result": {
    "content": [
      {
        "type": "text",
        "text": "[]"
      }
    ],
    "isError": false
  }
}
```

### `POST /mcp/lab/{tool_name}`

Forwards the JSON request body as MCP tool parameters to the `lab` source.

Idempotency: same rules as `/mcp/literature/{tool_name}`.

Request:

```json
{
  "relative_path": "sop/cell-assembly/index.md"
}
```

Response:

```json
{
  "result": {
    "content": [
      {
        "type": "text",
        "text": "# Cell Assembly SOP"
      }
    ],
    "isError": false
  }
}
```

If a source is unavailable, the gateway returns HTTP `200` with a degraded
payload instead of failing the entire request:

```json
{
  "degraded": true,
  "missing_source": "lab",
  "result": null
}
```

### `POST /mcp/query`

Runs the intent-aware federated query planner.

Allowed `mode` values are:

- `auto`: classify the user job and choose a route.
- `wiki`: wiki-only route.
- `vector`: evidence-vector route.
- `graph`: graph traversal route.
- `hybrid`: wiki, vector, and graph route.

Planner intents are user-job names: `reporting`, `know-how`, `insight`, and
`other`. They are distinct from backend route names.

Idempotency: safe to retry for the same corpus snapshot. The gateway does not
cache results in v1.0.0, so repeated calls may produce different prose if the
synthesis model is nondeterministic.

Request:

```json
{
  "query": "Draft a summary of LFP cathode degradation including our internal SOPs and reports",
  "mode": "auto",
  "quality": false
}
```

Response:

```json
{
  "answer": "LFP cathode degradation is associated with storage temperature and cycling protocol.",
  "intent": "reporting",
  "confidence": 0.92,
  "route": "hybrid",
  "citations": [
    {
      "source": "literature",
      "doc_id": "papers/sample-lfp-001",
      "chunk_index": 3,
      "page": null,
      "quote": "Capacity fade accelerates after high-temperature storage.",
      "confidence": 0.94,
      "ingested_at": "2026-04-28T10:30:00Z",
      "verify_against_source": true,
      "citation_type": "evidence"
    }
  ],
  "sources_consulted": ["literature", "lab"],
  "sources_unavailable": []
}
```

If one source is unavailable and the planner can still answer, the response
names the missing source in `sources_unavailable`. If no source can answer, the
gateway returns `503`.

## Streaming Behavior

`/mcp/query` returns one whole JSON response in v1.0.0. It does not stream model
tokens or citation updates.

Streaming is reserved for a future v1.x addition. If added without breaking the
existing response shape, it will use SSE with event names like this:

```text
event: answer.delta
data: {"text":"LFP degradation"}

event: citation
data: {"doc_id":"papers/sample-lfp-001","chunk_index":3}

event: done
data: {"sources_unavailable":[]}
```

## Timeout Semantics

Source subprocess crash recovery is handled by `MCPPool`. A permanently
unavailable source is reported as degraded rather than as a gateway crash.

In v1.0.0 the gateway does not persist an app-level timeout budget per source
call. Deployments should set reverse-proxy and ASGI server timeouts. Future v1.x
may add source-level timeout controls that return `408 source_timeout`.

Wiki write lock acquisition times out after 10 seconds and returns `503`.

## Out Of Scope For v1

- Multi-tenant permissions and per-user authorization beyond Cloudflare Access.
- A bearer-token fallback.
- Report templates, `cite_pack` tools, or report assembly. Writing apps own
  report structure.
- Pagination. v1 responses are whole-result payloads.
- Auto-generated SDKs.
- Streaming `/mcp/query` responses.
